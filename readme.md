## iblp的一些发现

IBLP有一些独有的现象，例如`(1,0)1(2,1,0)1(3,2,1,0)2(4,*3,2,1,0)2(5,4,3,2)2(6,5,4,1,0)2(7,6,5,4)2(8,7,6,1,0)2(9,8,7,6)2(10,9,8,1,0)2`，它是`(1,0)1(2,1,0)1(3,2,1,0)2(4,*3,2,1,0)2(5,4,3,2)2(6,5,4,1,0)2(7,6,5,4)2(8,7,6,1,0)2(9,8,7,6)2`的后继，表面上它可以展开，实际上却不能，也就是“坏图案”。

同样的，`(1,0)1(2,1,0)1(3,2,1,0)2(4,*3,2,1,0)2(5,3,2,1,0)3(6,*5,3,2,1,0)3(7,6,5,3)2(8,*7,6,3,2,1,0)4(9,*8,*7,6,5,3,2,1,0)5(10,9,6,5,3)3(11,*10,9,7,6,5,3)4(12,*11,*10,9,8,7,6,5,3)5(13,*12,*11,*10,9,8,7,6,5,3)5(14,10,6,5,3)3(15,*14,10,7,6,5,3)4(16,*15,*14,10,8,7,6,5,3)5(17,*16,*15,*14,10,9,8,7,6,5,3)6(18,17,*16,15,14,10,2,1,0)5`(1,1,1,1,2,2,0,0,2,0,1,2,0,1,2,1),尝试在iblp.py中输入operation 1的时候，就会爆出

```
Bad pattern: first copy is undefined. Returning E(A,0) (i.e. cut).
Applied cut operation.
```

原因是在执行`_copy_one_top_row_replace_last`的时候,`any(v is None for v in dst_row)`为true，展开不了似了，只能取后继。
